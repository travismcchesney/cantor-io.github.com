Adaptable.

Being adaptable solves issues before they become problems.

There are two aspects which allow Cantor to handle everything from tiny districts to those with hundreds of thousands of students. The first is the ability to shard integration executions into subgroupings of nearly any size. The other is a completely dynamic and horizontally scaling infrastructure, based upon demand.
Processing and data persistence nodes can exist on Cantor's cloud infrastructure, but can also be hosted on locally owned and maintained district infrastructure. In the latter setup, district data at every point never need leave the confines of your own datacenter. This includes logs and credentials.
Every district does things, sometimes dramatically, differently from one another. Cantor's model is based around event stream processing which allows for lenses to be applied at multiple points. Via a simple Domain Specific Language, complex business rules can be defined and applied to your district's integration.

Secure.

Security is a foremost concern, not an afterthought.

There is complete data and process isolation between districts. This is true whether districts leverage our infrastructure or their own. Linux containers are dynamically allocated, created, process said integration, and are then completely destroyed.
Credentials and sensitive information are always salted/hashed via SHA-512 or encrypted via AES-512. Districts may also optionally leverage adaptive ciphers like bcrypt to increase the computational power needed for brute-force search attacks into the future.
All traffic is cryptographically secured. This includes not only the communications to/from each educational system, but also within the Cantor infrastructure itself. If completely unavailable for a third-party system, the district is informed and consent required.

Friendly.

Open and everywhere.

We are big believers in eating your own dog food. Our web, mobile, iPhone, and Android applications all consume our openly available RESTful API, secured via oAuth. This allows districts to perform all Cantor administration operations programatically.
Cantor provides district administrators with robust web and mobile applications that allow them to view logs, data integrity alerts, manually trigger integrations, change settings, and more. Free iPhone and Android applications are also in development, allowing Cantor to be in your pocket at all times.
Each new feature added, integration module created, and document updated is completely free and available to districts. We are also big believers in open source development and contribute as often as possible.

Smart.

Work smart, not hard.

Mock system integration capabilities are available to test custom business rules or general questions you might have. No longer do you have to wonder "what will the data look like if X happens". You can try it. Right now.
To help ensure only valid data is integrated, Cantor performs checks of common data integrity issues prior to execution (e.g. staff accidentally rolled-forward in January). Cantor is also defensively programmed, helping to ensure that newest update you took doesn't interfere with the integration itself.
Cantor leverages many tried and true mathematical concepts, is developed in a statically type/purely functional manner, has complete unit/integration test coverage, is completely automated at all levels, and is self documenting.
